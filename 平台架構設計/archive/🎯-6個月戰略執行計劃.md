# 🎯 6 個月戰略執行計劃

> **團隊**：1 資深（Young）+ 2 超級 Junior 高中生（張靖杭、啟翔）
> **目標**：滿足方大哥期待 + 做到規模化 + All-in-one 教育平台
> **時間**：6 個月（24 週）
> **策略**：有意義、有謀略、可達成

---

## 🔍 深度思考：核心問題分析

### 問題 1：團隊能力不對稱

**現實**：
- Young：資深工程師，懂 AI、產品、架構、線上教育
- 高中生 × 2：程式基礎、學習力強，但不懂 CI/CD、GCP、Modern 開發

**錯誤做法**：
```
平凡 PM：把工作平分
→ Young 做 1/3，高中生各做 1/3
→ 結果：高中生卡住，進度延遲，Young 救火
```

**正確做法**：
```
戰略 PM：根據能力分配角色
→ Young 做「關鍵路徑」（技術風險高的）
→ 高中生做「平行任務」（有邊界、可學習的）
→ 結果：高中生邊做邊學，Young 專注核心
```

---

### 問題 2：6 個月要做什麼？

**方大哥的真實需求**（不是功能清單）：
1. **證明 AI 可以幫助學生學習閱讀**（核心價值）
2. **能在真實學校使用**（可用性）
3. **為未來規模化打基礎**（架構）

**錯誤理解**：
```
平凡 PM：6 個月做完所有功能
→ 朗讀分析、蘇格拉底對話、班級管理、作業系統、Agent 架構...
→ 結果：每個都做一半，沒一個能用
```

**正確理解**：
```
戰略 PM：6 個月做「階段性可用產品」
→ Week 10：核心功能可用（學生能練習）
→ Week 16：學習流程完整（學生 + 對話 + 歷程）
→ Week 24：規模化工具就緒（多校多班）
→ 結果：每個階段都有交付價值
```

---

### 問題 3：風險在哪？

**技術風險**：
1. **Whisper 兒童語音準確率**（Week 2 驗證，高風險）
2. **LLM 成本爆炸**（需要控制對話輪數）
3. **高中生學習曲線**（需要循序漸進）

**產品風險**：
1. **方大哥期待不明確**（需要早期 Demo 對齊）
2. **曾教授認可度**（5 次 Demo 都要過關）
3. **真實學校使用場景**（Beta 測試才知道問題）

**組織風險**：
1. **高中生可能放棄**（太難、太累、沒成就感）
2. **時程延遲**（排太滿、沒緩衝）
3. **品質低落**（趕進度、沒測試）

## 🎯 戰略核心：3 個關鍵決策

### 決策 1：核心價值優先（不是功能清單）

**問題**：6 個月內，什麼最重要？

**答案**：**Week 10 能 Demo 一個可用的核心功能**

```
核心功能 = 朗讀分析 + 即時回饋

為什麼這就夠了？
1. 證明 AI 可以幫助學生（方大哥要的）
2. 曾教授能評估教學效果（Demo 2）
3. 學生能實際使用（找學校試用）

其他功能：
- 班級管理？後面再說（Week 17-20）
- Agent 系統？驗證核心後再做（Week 13-14）
- 規模化？有用戶再考慮（Week 21-24）
```

**策略意義**：
- ✅ 降低風險（先做最重要的）
- ✅ 早期驗證（Week 10 就知道行不行）
- ✅ 保持士氣（高中生看到成果）

---

### 決策 2：讓高中生做「有邊界」的任務

**問題**：高中生不懂 CI/CD、GCP，怎麼辦？

**錯誤做法**：
```
強迫高中生學：
→ Week 1 教 Docker
→ Week 2 教 GCP
→ Week 3 教 CI/CD
→ 結果：學不會，浪費時間
```

**正確做法**：
```
讓高中生做「不需要懂這些」的任務：

Week 3-4：做 UI 元件（React）
  - 錄音按鈕
  - 播放器
  - 回饋介面
  → 不需要懂後端、部署

Week 7-8：做學生介面（Next.js）
  - 課文選擇頁面
  - 練習頁面
  - 學習記錄頁面
  → 只需要懂前端

Week 15-16：做資料視覺化（Recharts）
  - 學習曲線圖
  - 進度圖表
  → 有範例可參考
```

**策略意義**：
- ✅ 降低學習曲線（專注一個領域）
- ✅ 有成就感（看得到成果）
- ✅ Young 專注核心技術（Whisper、Agent）

---

### 決策 3：降低外部依賴風險

**問題**：Azure $250/月、LLM $45/月，如果不準或成本爆炸怎麼辦？

**策略**：
```
Week 2 立即驗證（技術去風險）：
1. Whisper 兒童語音測試
   - 如果準確率 >85% → 用 Whisper
   - 如果準確率 <85% → 切換 Google Speech

2. LLM 成本測試
   - 測試 50 學生 × 3 次練習 × 5 輪對話
   - 如果成本 >$50/月 → 用 Rule-based
   - 如果成本 <$50/月 → 用 LLM

3. 部署成本測試
   - GCP Compute Engine vs Cloud Run
   - 選成本低、穩定的方案
```

**策略意義**：
- ✅ Week 2 就知道技術可行性
- ✅ 有 Plan B（不會卡住）
- ✅ 成本可控（不會爆預算）
