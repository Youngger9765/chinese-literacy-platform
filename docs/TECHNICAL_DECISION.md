---

## 📌 兩大成功平台

本專案的技術決策基於以下兩個已驗證成功的平台：

1. **朗朗上口**（國語文閱讀流暢度分析系統）
   - 與曾世杰教授共同開發
   - 驗證了國語文朗讀分析的教學可行性
   - 提供完整的即時回饋與錯誤訂正機制

2. **Duotopia**（英文學習平台）
   - Young 獨立開發，3 個月完成
   - 已商業運作，獲高中與國中學生接受
   - 驗證了「校、班、生、師、課」架構的技術可行性
   - **功能完整**：朗讀練習、錯誤回饋、即時分析、單字學習全部已實作

## 🚀 開發策略：快速建置上線

### 為什麼不全部重新開發？

1. **⚠️ 教育平台建置經驗不足**
   - 兩位高中生對**教育平台建置的 Domain Know-how** 不完全清楚：
     - 「校、班、生、師、課」多層級獨立模組的架構設計
     - 體制內 vs 體制外的分模組模式
     - 教育產品的權限管理與使用情境
   - 可能沒有開發過這類多層級系統的經驗
   - 需要有經驗的人帶領，避免走冤枉路
   - 朗朗上口與 Duotopia 已踩過的坑，不需要再踩一次

2. **✅ 已有成功範例**
   - 朗朗上口證明：理論可以落地
   - Duotopia 證明：技術可以商用
   - 兩者結合 = 最快的成功路徑

3. **🎯 善用學生創意優勢**
   - 學生**較具有創意**，可把時間放在**更需要創意的學習模組開發**
   - 基礎建設（架構、權限、合規）已有業界慣例，直接使用既有範式
   - 讓學生發揮創意在有趣的部分：
     - 生字學習的互動設計
     - 蘇格拉底對話的提問邏輯
     - 學習遊戲化與激勵機制
   - 而非花時間在底層基礎建設（已有成熟解決方案）

### 開發優先順序

#### Phase 1（1-2 個月）：核心架構建立

**目標**：建立穩定的基礎平台

**借鑑 Duotopia 範式**：
- ✅ 校、班、生、師、課多層級資料模型
- ✅ 教師管理介面（班級、學生、課程管理）
- ✅ 學生登入與課程分配
- ✅ 權限管理機制
- ✅ 合規機制（隱私、資料保護）
- ✅ Azure Speech SDK 整合（設定為 zh-TW）

**技術棧建議**：

**Web 優先策略**（推薦）
- 後端：FastAPI / Node.js + Express
- 前端：Next.js（教師端 + 學生端統一使用 Web）
- 資料庫：PostgreSQL + Redis
- 語音：Azure Speech SDK (zh-TW)
- 部署：Vercel / AWS / GCP

**為什麼 Web 優先？**
- ✅ 開發簡單：一套程式碼，全平台通用
- ✅ 學習曲線低：兩位高中生更容易上手
- ✅ 維護容易：不需要維護多個平台版本
- ✅ 快速迭代：改一次就全平台更新
- ✅ 跨裝置：電腦、平板、手機瀏覽器都可用
- ✅ 未來彈性：需要時可用 PWA 或 Capacitor 轉 iOS/Android

**若未來需要原生 App**：
- 選項 1：PWA (Progressive Web App) - 免重寫，直接封裝
- 選項 2：Capacitor - Web 轉原生，保留 Web 程式碼
- 選項 3：Flutter - 需重寫，但效能最佳（最後選擇）

**交付成果**：
- 教師可建立學校、班級、學生
- 學生可登入並查看課程
- Azure Speech SDK 繁體中文語音識別測試通過

---

#### Phase 2（3-4 個月）：國語文學習模組

**目標**：實作國語文專屬功能

**借鑑朗朗上口設計**：
- ✅ 朗讀錄音與流暢度分析
- ✅ 即時回饋系統（六階段）
  - 朗讀結果總覽
  - 錄音播放與轉錄
  - 逐句分析對比
  - 錯字詞練習清單
  - 練習建議
  - AI 詳細分析
- ✅ 錯誤偵測與分類（跳字、加字、讀錯）
- ✅ 統計報告與學習追蹤

**新增功能**：
- 🆕 **生字學習模組**
  - 字型：部件拆解、結構分析
  - 字音：注音符號、發音練習
  - 字義：解釋、例句
  - 筆順：動畫示範
  - 書寫：平板手寫辨識（Google ML Kit）

- 🆕 **蘇格拉底對話系統**
  - LLM 驅動（OpenAI GPT-4o / Claude 3.5 Sonnet）
  - 語音輸入支援
  - 理解力評估
  - 對話歷史紀錄

**交付成果**：
- 學生可完整進行朗讀練習並獲得即時回饋
- 生字學習模組可獨立使用
- 蘇格拉底對話系統可評估理解力

## ⚠️ 風險管理

### 風險 1：Azure Speech SDK 中文準確度不足

**機率**：低
**影響**：高

**應對**：
- Phase 1 第一週立即測試繁體中文語音識別
- 準備備選方案（Google Speech-to-Text）
- 與曾教授確認最低可接受準確率標準（建議 > 90%）

### 風險 2：學生對 Domain Know-how 理解不足

**機率**：中
**影響**：中

**應對**：
- Young 每週 Code Review 與指導
- 提供朗朗上口功能說明文件
- 雙週與曾教授對齊確認方向
- 遇到教學理論問題立即詢問曾教授

### 風險 3：開發進度落後

**機率**：中
**影響**：中

**應對**：
- 敏捷開發，每 2 週交付可展示功能
- Phase 1 完成後評估進度，必要時調整 Phase 2 範圍
- 核心功能優先（朗讀分析 > 生字學習 > 蘇格拉底對話）

### 風險 4：使用者接受度低

**機率**：低（有朗朗上口與 Duotopia 驗證）
**影響**：高

**應對**：
- Beta 測試階段密集收集回饋
- 快速迭代 UI/UX
- 提供教師訓練與技術支援
