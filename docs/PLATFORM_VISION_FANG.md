## 說明

這個檔案記錄方大哥目前對「閱讀理解平台」的一些想法。

過去做「學國語」時，花最多時間在 **STT（語音辨識）** 跟 **注音符號** 上。因為 STT 會在朗讀扮演重要角色，做了一個程式來 Demo「溫暖但堅定」的助教（見「逐段朗讀」段落）。

已經把「學國語」的注音符號功能 port 到這平台。使用者只要點「注音 On」或「注音 Off」就可以打開或關掉注音符號。GitHub 已經更新，最新版本為 0.1.0。

## 角色說明

### 老師

- 用「單一帳號」註冊，email 確認，同意使用條款後，就可以開始使用
- 如果這是該校第一個老師註冊，系統會建立學校資料
- 校長有特別權限嗎？例如，校長能看到全校資料嗎？**此部分待未來再考慮**
- 老師要自己創建班級，加入學生。學生名單可以用 CSV 檔案匯入
- 一位老師可以有多個班級，每個班級有多個學生
- 加入學生時，只需要提供姓名及 email。學生的 email 也是用「單一帳號」，要跟老師同一個學校
- 老師需要上傳教材，包含課文、作者生平、課文簡介。每次上傳教材時，都要再次確認著作權
- 老師可以看到每一個學生的進展（學生進展細節待定）

### 學生

- 用「單一帳號」登入後，自動找到他的老師，自動進入班級
- 老師如果沒有加學生，這位學生就無法登入
- 平台設計給已經進入「讀而學」（read to learn）階段的學生，通常是三年級，但是也可能是國中生
- 他們可能缺乏學習興趣跟信心，需要鼓勵及引導

## 平台設計理念

- 平台要扮演一個「**溫暖但堅定**」的「**蘇格拉底**」助教
- AI 會用在「課文理解」及「出場卷」。其他流程都在程式內完成
  - 嚴格說，STT/TTS 也有用到 AI，不過這兩功能已經標準化，不需要自己用 API 呼叫 AI

## 學生學習流程（六步驟）

學生登入後，會看到老師這個學期已經上傳的課文清單。學生選課文後，進入閱讀六步驟：

### 1. 簡介

- 系統簡單介紹作者、這篇文章、相關背景（資料由老師提供）
- 學生可以點選 Mic，讓系統用口語說明，或自己看文字

### 2. 逐段朗讀

因為下列因素，STT 無法完美的轉錄語音：

- 學生發音不準或漏字
- 中文裡有太多同音異義的字跟詞，容易造成錯誤
- 雜訊干擾，降低識別率
- STT 有時難以判斷（以 iPhone 為例，它的 STT 已經非常好，但是還是無法 100% 正確）

因為平台要扮演「溫暖但堅定」的助教，做了幾個設計決定：

- **同音自動修正**：在逐段朗讀時，因為平台有文本，知道正確的文字，若遇到 STT 轉錄的字同音但不同調或不同字，平台會自動更改為正確的字，減少學生的挫折感
- **分級鼓勵機制**：
  - ≥ 80% 字跟原文一樣 → 給「非常好」的鼓勵，繼續下一句
  - ≥ 60% 字一樣 → 稱讚學生，繼續下一句
  - < 60% → 請學生再努力朗讀同一段文字

> Demo 程式：https://github.com/Shinjou/lingoleap-ai-reading-tutor
